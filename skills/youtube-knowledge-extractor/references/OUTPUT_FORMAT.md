# Output format

Use these headings in order after sufficient source content is available. Omit only empty interest subsections, never top-level sections. For inaccessible URL-only input, do not use this report format; request source content as directed by `SKILL.md`.

## 【影片一句話結論】

State the central idea in one sentence.

## 【3 分鐘重點摘要】

Provide concise thematic bullets rather than a timeline.

## 【對我有用的重點】

Use only applicable subsections: AI / 自動化、天文攝影、IT / 管理、影像創作、投資 / 商業趨勢. If none apply, write `普通，可略過。`

## 【可執行行動】

List three to seven source-supported actions when justified. If fewer are useful, explain why.

## 【值得存成知識卡】

Include 標題、核心概念、為什麼重要、應用場景、注意事項、關鍵詞. If no durable concept exists, write `不建議建立知識卡。`

## 【知識連結（Knowledge Graph）】

List each related topic and one brief reason for the connection.

## 【可以交給 AI Agent 的工作】

Use these subsections in order:

- 適合 Codex
- 適合 Hermes Agent
- 適合 OpenCode
- 適合 Claude Code

Assign work only when the named tool's relevant capabilities and required access are established. Write `無。` under a tool when no suitable work is warranted.

## 【需要查證的說法】

For each claim, provide 影片中的說法、為何需要查證、建議查證來源. Write `無明顯需要額外查證的說法。` when appropriate.

## 【整體評價】

- 影片價值：★☆☆☆☆ 至 ★★★★★
- 是否值得完整觀看：是 / 否
- 是否值得收藏：是 / 否
- 是否值得建立知識卡：是 / 否
- 是否適合分享給同事：是 / 否

Explain the rating in two to five sentences.
