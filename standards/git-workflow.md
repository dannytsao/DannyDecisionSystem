# Git Workflow

- Keep `main` stable and reviewable.
- Use one focused branch per task, prefixed with `codex/` when Codex creates it.
- Keep commits atomic and describe the user-visible outcome.
- Preserve unrelated worktree changes.
- Do not let multiple agents edit the same file concurrently.
- Validate affected Skills and workflows before committing.
